## **§2 — EXECUTIVE OVERVIEW & PLATFORM PURPOSE** 

RyskNode is an AI-driven trade-credit intelligence platform for Indian B2B Sellers making real-time, defensible credit decisions on Buyers. Six government data sources (MCA, GST, eCourts, EPFO, RBI, MCA Charges/ROC) feed a 35-column EntityProfile (RiskDNA). The Pralyon Tri-Core™ ensemble plus Signal Layers produce a Pralyon Risk Score (300–900), a RiskBand, a Confidence Signal, and a sanctioned credit limit via AnchorEngine™ — with full ShockFrame™ stress testing and TraceLayer™ explainability.

## **§6 — INTEGRATION BRIDGE — SIGNAL LAYERS** 

Source: integration/bridge.py | Entry function: enrich_feature_row(row, resolver_result, source_freshness_days) → dict `enrich_feature_row(row: dict, resolver_result: dict, source_freshness_days: int) → dict │ ├── _derive_ratios(row) │   current_ratio    = current_assets / current_liabilities │   quick_ratio      = (current_assets − inventory) / current_liabilities │   debt_to_equity   = total_debt / networth │   dso              = (receivables / revenue) × 365 │   turnover_cagr    = (revenue_y1 / revenue_y3)^(1/2) − 1 │ ├── _build_normalized_record(row)  ← injects reporting_period [v1.2 fix] │ ├── Signal Layer 1: compute_financial_score()   ← clamp() [v1.2] ├── Signal Layer 2: compute_identity_score() ├── Signal Layer 3: compute_legal_score()       ← INVERTED in Governance Score └── Signal Layer 4: compute_documentation_score() → Enriched RiskDNA (35 cols + 4 Signal Layer scores + derived ratios)` 

## **6.1 Signal Layer 1 — Financial Score (Weight: 40%)** 

|**Dimension**|**Weight**|**Metric**|**Scoring Thresholds**|
|---|---|---|---|
|Liquidity|25%|CurrentRatio|≥2.0→100 |1.5–2.0→80 |1.0–1.5→60 |<1.0→20|
|Leverage|20%|Debt-to-Equity|≤0.5→100 | 0.5–1.0→80 |1.0–2.0→55 |>3.0→10|
|Revenue Growth|20%|Revenue CAGR|>15%→100 | 5–15%→75 | 0–5%→50 |negative→15|
|CashConversion|15%|DSO (Days)|<30→100 | 30–60→75 | 60–90→50 |>120→20|
|Solvency|10%|Working Capital|Positive+growing→100 |Negative→15|
|BusinessAge|10%|Vintage (years)|>7→100 | 3–7→75 |1–3→40 |<1→10|



All scores bounded [0, 100] via safe_math.clamp(). 

## **6.2 Signal Layer 2 — Identity Score (Weight: 20%)** 

|**Component**|**Weight**|**Description**|
|---|---|---|
|Entitynamematch|30%|Exact /fuzzymatchgradefrom resolver_result|
|GSTIN-PAN linkage|20%|PANconsistency acrossMCAand GST|

## **6.3 Signal Layer 3 — Legal Score (Weight: 25%, INVERTED)** 

⚠ **CRITICAL: Signal Layer 3 is INVERTED in the Governance Score formula. Higher raw legal score = more legal risk. Applied as (100 − Signal Layer 3).** 

|**Dimension**|**Weight**|**Scoring Logic**|
|---|---|---|
|Totalcaseload|30%|0→0 |1–2→15 | 3–5→40 |>10→85|
|Active case severity|30%|DRT/NCLT → near 100;HighCourtheavilyweighted|
|Criminalexposure|25%|Any criminalcase→ rawscore 90|
|Caserecency|15%|Last12 months casesweighted 3×older|



## **6.4 Signal Layer 4 — Documentation Score (Weight: 15%)** 

|**Component**|**Weight**|**Scoring Logic**|
|---|---|---|
|Source coverage|35%|4+sources→100 | 3→80 |2→55 |1→25|
|Datafreshness|25%|<7d→100 |7–30d→85 | 30–90d→65 |>180d→30|
|GST filingregularity|25%|REGULAR→100 |MINOR_GAPS→70 |MAJOR_GAPS→30 |NON_FILER→0|
|Financial data depth|15%|3yrs→100|2→65|1→30|



## **6.5 Governance Score Formula** 

```
Governance Score = (0.40 × Signal Layer 1)
               + (0.20 × Signal Layer 2)
               + (0.25 × [100 − Signal Layer 3])
               + (0.15 × Signal Layer 4)
```

|5|Sort by observation_date|Temporalsort— nofuture-dataleakage|
|6|Train/Val/Test70/15/15|Strictly ordered;no shuffle|
|7|Fit WoETransformer on train<br>only|10 quantile bins; WoE ∈ [−4, +4]; min bin rate 0.001|
|8|Transform val + test|No re-fittingon held-out sets|
|9|Return DataSplit|X/y allsplits+ WoEvariants+ woe_encoder +default_rate|



## WoE Formula: 

```
pd.qcut(train_col, q=10, duplicates='drop')    ← quantile binning
WoE = log(dist_non_events / dist_events)       ← per bin, clipped [−4, +4]
Categoricals: per-category WoE dict
Unseen at inference: WoE = 0.0
Output col names: <feature>_woe  (e.g. current_ratio_woe)
```
