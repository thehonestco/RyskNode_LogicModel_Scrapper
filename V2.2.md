## **§2 — EXECUTIVE OVERVIEW & PLATFORM PURPOSE** 

RyskNode is an AI-driven trade-credit intelligence platform for Indian B2B Sellers making real-time, defensible credit decisions on Buyers. Six government data sources feed a 35-column EntityProfile (RiskDNA). The Pralyon Tri-Core ensemble plus Signal Layers produce a Pralyon Risk Score (300-900), RiskBand, Confidence Signal, and sanctioned credit limit via AnchorEngine — with ShockFrame stress testing and TraceLayer explainability.

|---|---|---|

## **§5 — PART 1: DATA INTELLIGENCE LAYER** 

Part 1 fetches raw data from 6 government/regulatory APIs via an Aggregator SDK, extracts structured fields, resolves conflicts, builds the NormalizedRecord, derives 35 engineered features (RiskDNA), and runs the 5-signal BehaviourPrint chain to produce a FinalFeatureRow ready for Part 2. 

## **5.1  Pipeline Flow (part1/pipeline.py)** 

```
score_entity(entity_id, seller_id, config)
 └─ _fetch_all_sources(entity_id)       # parallel Aggregator SDK calls
 └─ _run_extractors(raw_payloads)       # source-specific extract_*() functions
 └─ _resolve_conflicts(extracted_dicts) # priority: MCA > GST > Udyam > eCourts
 └─ NormalizedRecord(**merged_dict)     # Pydantic v2 validation
```

```
 └─ feature_builder.build(norm_rec)     # 35-col FeatureRow with engineered ratios
 └─ ConductScoringChain.score(norm_rec) # BehaviourPrint conduct_score 0-100
 └─ bridge.enrich_feature_row(fr)       # Signal Layers 1-4 → FinalFeatureRow
```

## **5.2  Source Priority for Conflict Resolution** 

|**Priority**|**Source**|**Preferred For**|
|---|---|---|
|1(highest)|MCA|Financial data,incorporation date,CIN,entity_type|
|2|GST|GSTIN, gst_status,filinghistory,state|
|3|Udyam|MSME category,PAN(corroborating),legal_name fallback|
|4|eCourts|Legal case counts(exclusive source)|
|5|MCA Charges|Charge counts(exclusive source)|

## **5.3  BehaviourPrint — 5-Signal Conduct Scoring Chain** 

File: part1/conduct/conduct_scoring_chain.py | Class: ConductScoringChain 

Produces conduct_score (0-100). Each signal is scored 0-100 and weighted per below. 

|**Signal**<br>**#**|**Signal Name**|**Weight**|**Input(s) Used**|**Scoring Logic**|
|---|---|---|---|---|
|1|GST Filing Consistency|0.30|gst_filing_consistency_ratio|0.95+=100 | 0.80-0.95=75 | 0.50-<br>0.80=40|<0.50=0|
|2|Financial Statement Filing (MCA)|0.20|incorporation_date vs<br>financialData completeness|All 5 yrs=100 | 3-4=75 | 1-2=40 |<br>0=0|
|3|Legal Conduct|0.25|criminal_case_count,<br>pending_case_count|0 criminal+0 pending=100 |<br>criminal>0=0 | pending 1-2=60 |<br>>2=20|
|4|Charge Regularity|0.15|old_unsatisfied_charge_count,<br>has_recent_charge_90d|0 old unsatisfied + no recent=100;<br>deductionsper charge|
|5|EPFO/PF Filing|0.10|pf_filing_regular(bool)|True=100|False=0|



Formula: conduct_score = sum(signal_score_i x weight_i) clamped to [0, 100] 

## **§6 — INTEGRATION BRIDGE — SIGNAL LAYERS (integration/bridge.py)** 

The Bridge transforms the raw FeatureRow from Part 1 into an enriched FinalFeatureRow by computing 4 domain-specific Signal Layer scores (0-100 each) and injecting reporting_period. All 4 signal scores plus conduct_score become input features to the Pralyon Tri-Core ensemble. 

## **6.1  Signal Layer Definitions & Formulas** 

|**Signal**<br>**Layer**|**Name**|**Weight in**<br>**Bridge Score**|**Key Inputs**|**Logic**|
|---|---|---|---|---|
|SL1|Financial Health|0.40|current_ratio, debt_to_equity,<br>net_revenue_cagr_5y, dso|Weighted sub-scores: liquidity<br>0.30 + leverage 0.25 + growth<br>0.30 + efficiency0.15|
|SL2|Identity & Compliance|0.25|gst_status, pan_match (MCA vs GST),<br>incorporation_date validity,<br>udyam_no|gst_active=50pts;<br>pan_match=25pts;<br>valid_date=15pts;<br>udyam_present=10pts|
|SL3|Legal Risk|0.20|criminal_case_count,<br>case_count_active, case_count_drt,<br>has_insolvency|Base 100 minus deductions:<br>criminal 15/case, DRT 10/case,<br>NCLT=full deduct,active 3/case|
|SL4|Documentation|0.15|sources_available (len),<br>mca_pdf_required, conflict_flags (len)|>=4 sources=100 | 3=75 | 2=50<br>| <2=25; -10 per conflict_flag; -<br>20 if mca_pdf_required|



## **6.2  Overall Bridge Score Formula** 

```
bridge_score = (
    SL1_score * 0.40 +
    SL2_score * 0.25 +
    SL3_score * 0.20 +
    SL4_score * 0.15
)
# Clamped to [0, 100]; NOT a separate output — feeds into FinalFeatureRow as
# financial_score, identity_score, legal_score, documentation_score columns
```

## **7.6  ShockFrame — 5 Stress Scenarios (StressTester)** 

File: part2/scoring/stress_tester.py | Output: List[ScenarioResult] (5 items) 

|**Scenario**|**PD Shock**|**LGD Shock**|**Revenue Shock**|**RBI Alignment**|
|---|---|---|---|---|
|base|+0.00|+0.00|+0.00|Current conditions|
|mild|+0.05|+0.05|−10%|Mild economic slowdown|
|moderate|+0.10|+0.10|−20%|Moderate stress|
|severe|+0.20|+0.15|−35%|Severe downturn/IBC wave|
|extreme|+0.35|+0.20|−50%|Systemic crisis(RBI DSCR 0.85 scenario)|



```
# Per scenario:
```

```
stressed_pd  = min(1.0, base_pd + pd_shock)
stressed_lgd = min(0.95, base_lgd + lgd_shock)
stressed_rev = net_revenue_latest * (1 + rev_shock)
```

```
EL  = stressed_pd * stressed_lgd * exposure_amount    # Expected Loss
```

```
UL  = sqrt(stressed_pd * (1-stressed_pd)) * stressed_lgd * exposure_amount  # Unexpected Loss
NPA_flag = stressed_pd > NPA_THRESHOLD (0.15 hardcoded)
```
